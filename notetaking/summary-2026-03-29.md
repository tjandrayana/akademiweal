Session notes: shipped features, fixes, and a practical backlog for what to improve next.

## Done — 2026-03-30 (UI redesign + gamification fixes)

### Map & lesson UI overhaul
- **Home duplicate HUD** removed — `hideHud` prop added to `MapScreen`; Home now owns the only HUD bar.
- **Zone scenery v2** — all 8 zones redrawn with colorful SVG gradients, animated clouds/coins/flags (8 `iq-` keyframes); `ZoneTrailPath` replaces static polyline with per-zone bezier paths and `iq-path-dash` animation.
- **Zone 1 Penny Plains** fully redrawn — kawaii sun, animated clouds, flowers, START sign, rainbow arches.
- **LessonIntro redesign** — zone-aware SVG background, MascotEvolution mascot in bouncing circle, streak badge, tip card (💡 hook), content card (📖 body), fun fact strip, green CTA.
- **Quiz redesign** — `ZoneLessonScene` per-zone backgrounds, A/B/C/D list answers with colored letter badges, feedback banner (correct/wrong), XP popup overlay (`iq-xp-pop`).

### Gamification / integrity fixes
- **P0: Shuffle answer options** — Fisher-Yates shuffle via `useMemo` keyed on `lesson.id`; correct answer compared by value, not index; stable within a lesson attempt.
- **P0: Guest→login XP in HUD** — After `migrateGuestProgressToUser`, `mergeServerXp(data.xp)` wouldn't fire `XP_UPDATED_EVENT` when server XP matched local merged value; now always dispatches the event after migration so any mounted HUD refreshes.
- **Mascot tier mismatch fixed** — Lesson/quiz now uses `getDisplayMascotEvolutionLevel(xp, pathStep)` (same path-cap formula as Home) instead of uncapped `getMascotEvolutionLevel(xp)`, ensuring Lesson and Home always show the same tier.
- **Mascot card redesign** — quiz mascot replaced with a profile-style dark card: 88px, tier-colored border + glow ring + gradient background + animated tier-name strip; animation driven by `mascotTier.anim`.

## Done (high level)

### Authentication & users
- **Email + password** registration and login (`POST /register`, `POST /login`), bcrypt-backed `password_hash`, migration `012_user_password.sql`.
- **Legacy rows** without a password can **claim** an account via register with the same email.
- JWT flow unchanged; login/register responses include **`xp`** for client merge.

### Progress & storage
- **Per-user localStorage** namespaced as `akademiweal_u{userId}_*` (from JWT `sub`); guests use **`akademiweal_prelogin_*`**.
- **Migrations on login**: legacy flat keys → namespaced; **guest prelogin → user** merge on sign-in (`migrateGuestProgressToUser`).
- **XP sync** to server (`/me/sync`); **`syncXpTotal`** no-ops when not logged in.

### Guest “try before login”
- Configurable via **`VITE_GUEST_MAX_FREE_PATH_STEP`** in frontend `.env` (default 5, clamped 1–80).
- Map and lesson flows **block** steps beyond the free limit; nodes show locked on map for guests past the cap.
- **`/sambung-belajar`** — playful **GuestUnlock** interstitial (animations, tap-to-play coin, CTAs to login/register) instead of jumping straight to the login form.
- **Routing**: most of the app is public; **`/profile`** stays behind auth.

### Leaderboard & XP (from earlier work in this arc)
- Server **`xp_total`**, **`GET /leaderboard`**, authenticated **`/me`**, **`POST /me/sync`**; frontend merge + **Peringkat** page.

### UX / gamification display
- **Mascot tier on map/profile** capped by **path step** so high server XP does not show **Toro** while still on Zona 1 (`getDisplayMascotEvolutionLevel`).

### Backend bugfix
- **Events insert**: pass JSON to PostgreSQL as **string**, not `[]byte` as bytea, to fix **`SQLSTATE 22P02`** on `metadata` JSONB.

## Improvements to consider next

### Personalization & identity (logged-in)
- **Show the user’s name in gameplay / near mascot** — Replace generic labels (e.g. **“Investor”** on **`Home`**) with something tied to the account: e.g. **email local-part** (`a` from `a@gmail.com`) or a future **`display_name`** from **`GET /me`**. Use consistently on **HUD**, **map shell**, and optional **Profile** header; respect **privacy** (truncate, avoid flashing full email). May need **`/me`** extended with `email` or `nickname`.

### Celebration & progression UX
- **Mascot evolution level-up screen** — When the player **levels up** in a way that **unlocks the next mascot tier** (e.g. Penny → Toro → … per `MascotEvolution.jsx` / `getMascotByLevel`), **do not jump straight back to the map**. Add a **full-screen, interactive celebration** in the same spirit as **`/sambung-belajar`**: playful, funny, **premium** (gold/navy/glass, motion, optional sound), and **friendly** copy (Indonesian). Ideas: tap-to-sparkle coin or mascot, short tier “reveal,” confetti/shine, silly one-liners, then a clear CTA (**Lanjut** / **Kembali ke peta**). **Detection**: compare previous vs new **`displayMascotEvolutionLevel`** (or tier boundaries from `TIERS`) after XP gains — e.g. on **`Result`** continue, after **`markLessonComplete`**, or when landing **`Home`** after session; persist **`lastCelebratedMascotTier`** in localStorage to avoid repeat spam.

### XP, guest merge & anti-abuse
- ~~**Guest → login: XP not carried into HUD**~~ — **Fixed (2026-03-30)**: `XP_UPDATED_EVENT` now dispatched after `migrateGuestProgressToUser` in `applyAuthPayload`.
- ~~**Replay XP farming**~~ — **Fixed (2026-03-30)**: Session snapshots `completedLessons` at level load; correct answers on already-completed lessons grant 0 XP and no popup; `XP_PER_LESSON_COMPLETE` bonus also skipped if no new lessons were completed this session.
- **Quiz: shuffle answer options** — Today the **correct answer often sits in a fixed position** (e.g. **always second**), so users memorize **slot** not content. **Shuffle** the **`options`** array **per lesson mount** or **per retry** (e.g. Fisher–Yates), and compare the user’s choice to the **canonical answer** (same label/index mapping). Retain accessibility (labels, `aria-posinset` if needed).
- **Two-device + guest-login “level” vs XP mismatch (repro)** — **Device A**: logged in as **`a@gmail.com`**, progress around **level ~11** (map step / mascot / curriculum — whichever the user meant), **XP 78**. **Device B**: **Guest**, finishes **5** free steps, then **logs into the same account**. On Device B: **“level” drops to ~6** but **XP still shows 78**. **Why**: **`xp_total`** is **per user on the server** and merges into local; **`completed_lessons` / map path** are still **mostly localStorage per browser**. Device B only has **guest merge + ~5 completions**, not Device A’s full path. **`getDisplayMascotEvolutionLevel`** = **`min(XP-tier, pathStep)`**, so a **short local path** **caps** mascot / perceived **“level”** while **coin XP** shows **server truth** — **confusing split**. **Fix**: **persist completions (and path) server-side** and **hydrate on login** / app start; or **temporary rule** when authenticated: if server XP implies farther progress than local, **prefer server snapshot** once available; document until API exists.
